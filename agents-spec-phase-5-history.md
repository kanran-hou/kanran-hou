 # Phase 5 — 历史记录与报告导出（P1）
 
 > **目标**: 实现历史分析记录存储与展示、支持选中历史文案二次分析、分析报告生成图片并导出分享。
 > **依赖**: Phase 3（前端页面框架） + Phase 2（分析接口）
 > **产出**: 历史记录页面 + 二次分析功能 + 报告图片导出
 
 ---
 
 ## 技术决策
 
 | 选项 | 选定 | 理由 |
 |------|------|------|
 | 本地缓存 | **wx.setStorageSync / uni.setStorage** | 微信小程序原生存储 |
 | 后端存储 | 复用 Phase 1 的 `copywriting_analysis` 表 | 统一数据源 |
 | 报告图片生成 | **wx.canvasToTempFilePath** + canvas 绘制 | 小程序原生方案 |
 | 图片保存 | **wx.saveImageToPhotosAlbum** | 微信小程序 API |
 
 ---
 
 ## 任务清单
 
 ### 5.1 历史记录模块
 - [ ] 设计历史记录列表页面（`pages/history/history`）
   - 展示历史分析记录列表（按时间倒序，分页加载）
   - 每条记录显示：文案摘要（前 30 字）、赛道标签、综合评分和评级、分析时间
 - [ ] 实现本地缓存存储策略
   - 分析完成后自动缓存到 `wx.setStorageSync('history', [...])`
   - 缓存上限 50 条，超出自动移除最旧记录
 - [ ] 实现后端历史记录查询 API
   - `GET /api/v1/history?user_openid=xxx&page=1&page_size=20`
   - `GET /api/v1/history/{analysis_id}` 获取单条完整记录
 - [ ] 实现历史记录详情页（点击列表项 → 查看完整分析结果）
   - 复用 Phase 3 的 `pages/result/result` 组件，传入 `analysis_id` 参数
 - [ ] 实现「二次分析」功能
   - 历史详情页提供「重新分析」按钮
   - 点击后跳转到首页，原文和赛道预填，用户可修改后重新提交
 
 ### 5.2 报告图片导出
 - [ ] 设计报告图片模板（移动端竖版布局）
   - 顶部：小程序品牌 Header + 文案摘要
   - 中部：综合评分环形图 + 5 维度雷达图（或柱状图）
   - 下部：核心优化建议摘要（精炼 3-4 条）
   - 底部：生成时间 + 「由 CopyMind AI 生成」水印
 - [ ] 使用 canvas 绘制报告图
   - 编写 `ReportCardPainter` 模块
   - 支持文本自动换行
   - 支持中文渲染
   - 支持圆角、渐变、阴影等视觉元素
 - [ ] 实现保存到相册功能
   - canvas → 临时文件 → `wx.saveImageToPhotosAlbum`
   - 获取相册授权（首次请求弹窗说明）
 - [ ] 实现分享功能（可选）
   - 生成图片后显示分享面板
   - 支持发送给微信好友/朋友圈

 ## 验收标准
 
 1. 分析完成后自动存入本地缓存 + 后端数据库
 2. 历史记录列表按时间倒序展示，每条显示摘要 + 评分 + 赛道 + 时间
 3. 历史列表支持下拉加载更多（分页）
 4. 点击历史记录项展示完整分析结果
 5. 「重新分析」按钮跳转到首页并预填原文和赛道
 6. 报告图片生成成功，包含评分 + 5 维度 + 优化建议摘要
 7. 保存到相册功能正常，图片清晰度可读
 8. 未授权相册权限时弹窗引导授权
