 # Phase 2 — AI 分析与 Prompt 引擎（P0）
 
 > **目标**: 实现大模型 API 调度、4 赛道 Prompt 规则库、JSON 结构化输出解析与校验，完成核心分析能力。
 > **依赖**: Phase 1（数据库 + 后端骨架）
 > **产出**: 可调用的 `/api/v1/analyze` 端点 + Prompt 规则库 + 结构化解析模块
 
 ---
 
 ## 技术决策
 
 | 选项 | 选定 | 理由 |
 |------|------|------|
 | 大模型 API | **DeepSeek Chat API**（首推） | 性价比高、中文能力强、JSON 输出稳定 |
 | 备选模型 | 通义千问 qwen-max / 智谱 GLM-4 | 容灾切换 |
 | API 调用库 | **httpx** | FastAPI 生态异步兼容 |
 | JSON 校验 | **Pydantic v2** | 强类型校验，与 FastAPI 深度集成 |
 | 失败重试 | **tenacity** | 指数退避重试 |

 ## Prompt 规则设计要点
 
 ### 通用约束模板（所有赛道共用）
 
 ```
 你是一个专业的文案分析助手。请严格按以下 JSON 格式输出分析结果。
 不要输出任何额外说明、解释、前缀或后缀。确保输出是合法的 JSON。
 评分均为 0-100 的整数。评级规则：综合分≥85为S(爆款潜力)，
 ≥70为A(中等)，≥50为B(待优化)，<50为C(低分)。
 ```
 
 ### 赛道差异化要点
 
 | 赛道 | 标题分析侧重 | 情绪侧重 | 卖点结构侧重 |
 |------|-------------|---------|-------------|
 | 小红书种草 | 表情符号、数字、"绝了"类感叹词 | 共情/向往/焦虑 | 场景化卖点、使用体验 |
 | 电商商品 | 利益承诺、紧迫感、"限时"类词 | 信任/紧迫 | FAB法则、促单话术 |
 | 本地文旅 | 地名、打卡、季节限定词 | 探索欲/归属感 | 路线/价格/体验组合 |
 | 短视频 | 前 3 秒钩子、悬念、反问 | 高密度情绪起伏 | 节奏密度、完播率设计 |

 ## 验收标准
 
 1. `POST /api/v1/analyze` 传入有效文案返回完整 5 维度 JSON
 2. 传入 4 种赛道参数均能返回赛道特色分析（可验证差异）
 3. 传入超过 5000 字文案返回 413 状态码 + 错误提示
 4. 传入空文本返回 400 状态码
 5. 模拟 API 超时场景，自动切换到备用 Provider
 6. 连续 3 次解析失败后返回降级错误信息
 7. 分析结果正确写入 `copywriting_analysis` 表
 8. 行为日志同步写入 `users_behavior_log` 表
 9. 输出优化建议包含至少 3 套改写标题 + 正文段落调整方案
 10. 优化建议中每条附带原文位置标注和对应分析依据
