 # Phase 4 — RAG 知识库与 OCR（P1）
 
 > **目标**: 构建轻量级 RAG 向量知识库，实现同赛道高分文案匹配；集成 OCR 图文识别能力，支持图片文案自动转文本。
 > **依赖**: Phase 2（AI 引擎可用） + Phase 3（前端录入页预留图片上传入口）
 > **产出**: 知识库检索模块 + OCR 识别模块 + 优化后的分析流程
 
 ---
 
 ## 技术决策
 
 | 选项 | 选定 | 理由 |
 |------|------|------|
 | 向量库 | **Faiss（本地）** | 轻量无运维、1000+ 样本量级足够 |
 | 文本嵌入 | **text2vec-base-chinese**（HuggingFace） | 中文语义理解良好、本地可运行 |
 | OCR 引擎 | **PaddleOCR** | 中文识别精度高、支持水印过滤 |
 | 原始数据存储 | MySQL + JSON 字段 | 与 Phase 1 数据库统一管理 |
 
 ---
 
 ## 任务清单
 
 ### 4.1 知识库数据准备
 - [ ] 设计知识库表结构（`copywriting_templates` 表）
 - [ ] 字段：`id`, `track_type`, `title`, `content`, `tags`, `overall_score`, `source`, `created_at`
 - [ ] 收集/构造 1000+ 条覆盖 18 个内容品类的爆款文案样本
   - 小红书：美妆/穿搭/家居/美食/旅行/母婴/健身/读书
   - 电商：服饰/数码/食品/美妆/家居
   - 本地文旅：景点/探店/酒店/民宿
   - 短视频：剧情/知识/产品/生活方式
 - [ ] 编写数据导入脚本（批量插入 `copywriting_templates` 表）
 - [ ] 编写文本嵌入脚本（对每条文案生成 768 维向量，存储为 Faiss index）
 - [ ] 实现 Faiss index 构建与序列化（保存到本地文件）
 
 ### 4.2 向量检索服务
 - [ ] 实现 `VectorSearchService` 类
   - `build_index()`: 从数据库读取文本 → 生成嵌入 → 构建 Faiss index
   - `search(query, track_type, top_k=5)`: 按赛道过滤后返回最相似文案
   - `refresh_index()`: 增量更新 index（新文案入库后触发）
 - [ ] 实现 `TemplateService` 类
   - `get_top_templates(track_type, limit=5)`: 返回同赛道高分模板（用作 Prompt 参考上下文）
 - [ ] 缓存 Faiss index 到内存（服务启动时加载，周期性刷新）
 
 ### 4.3 知识库与 AI 分析联动
 - [ ] 修改 `POST /api/v1/analyze` 流程：用户提交文案后
   - 步骤 1：向量检索匹配 3-5 条同赛道高分案例
   - 步骤 2：将匹配结果作为参考上下文注入 Prompt
   - 步骤 3：大模型参考高分案例特征输出分析
 - [ ] 在分析结果中添加「参考案例」字段（展示匹配到的模板标题和核心特征）
 - [ ] 实现「查看参考案例」功能（用户可一键阅读同赛道高分文案）
 
 ### 4.4 OCR 图文识别
 - [ ] 搭建 PaddleOCR 轻量服务（或通过 API 封装调用）
 - [ ] 封装 `OCRService` 类
   - `recognize(image_bytes)`: 输入图片二进制 → 输出识别文本
   - `post_process(text)`: 过滤水印文字、无关字符、多余换行
 - [ ] 实现 `POST /api/v1/ocr/recognize` 端点
   - 请求体：`multipart/form-data`，字段 `image`
   - 响应体：`{ "text": "...", "word_count": 123 }`
 - [ ] 前端对接 OCR API（首页图片上传按钮 → 调用 OCR → 填充到输入框）
 - [ ] 实现图片上传前压缩（降低传输耗时，限制单张 < 5MB）
 
 ---
 
 ## 验收标准
 
 1. 知识库数据导入成功，`copywriting_templates` 表包含 1000+ 条数据
 2. 向量索引构建成功，同赛道相似文案检索 Top-5 结果语义合理
 3. 调用分析 API 时，知识库匹配结果成功注入 Prompt（可通过日志验证）
 4. 分析结果中「参考案例」字段返回正确的模板标题和核心特征
 5. OCR 接口识别图片中的中文文字，准确率 > 90%（非手写体）
 6. OCR 后处理模块正确过滤常见水印文字（"小红书"、"转载请注明出处"等）
 7. 前端图片上传 → OCR 识别 → 填充文本输入框全链路跑通
 8. 上传非图片文件返回 400 错误提示
