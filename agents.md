 # AI文案智能分析小程序 — 项目代理主控文档
 
 > **项目代号**: CopyMind
 > **本文档作用**: 全局项目编排，定义阶段划分、各阶段目标、进入/退出标准、以及每个阶段应交付的产物。配合各阶段独立 spec 文件（`agents-spec-phase-*.md`）使用。
 > **状态管理**: 每个阶段由 `agents.md` 中的 "当前阶段" 标记 + 对应 spec 文件中 checklist 共同维护。
 
 ---
 
 ## 项目概览
 
 面向自媒体、电商、本地文旅内容创作者的轻量化文案诊断工具。用户粘贴/上传文案后，调用大模型 API 完成多维度结构化分析，输出量化评分、问题标注、可直接落地的文案优化方案。基于 Vibe Coding 快速完成小程序前后端开发。
 
 **技术栈（初步选定）**：
 - 前端：微信小程序（原生 WXML/WXSS/JS 或 Uniapp）
 - 后端：Node.js (Express/Koa) 或 Python (FastAPI)
 - 数据库：MySQL + 向量库（Milvus / Faiss）
 - AI：国内合规大模型 API（DeepSeek / 通义千问 / 智谱等）
 - OCR：集成图像文字解析接口

 ## 阶段进入/退出规则
 
 ### 阶段推进原则
 
 1. 每个阶段必须**串行完成**，前一阶段退出后才能进入下一阶段。
 2. 阶段退出标准：该阶段 spec 中所有 P0 级 checklist 项标记为 `[x]` 完成。
 3. 若阶段包含 P0/P1 混合任务，P0 必须全部完成，P1 可部分完成但须在 spec 中明确标记遗留项。
 4. 每次进入新阶段前，执行一次轻量回归确认：上游阶段产物未发生破坏性变更。
 
 ### 阶段切换操作
 
 当需要进入下一阶段时：
 1. 更新 `agents.md` 中 "当前阶段" 标记
 2. 生成该阶段初始 checklist
 3. 创建或解压缩该阶段所需初始文件

 ## 团队指令
 
 > 以下指令供后续 AI 代理在执行各阶段任务时参考。
 
 ### 通用规范
 
 1. 所有代码输出前检查与需求文档的匹配度，标注偏离项。
 2. 环境变量、密钥等敏感信息统一放在 `.env` 文件，不硬编码。
 3. 每次提交须附带 changelog 条目（格式：`[Phase X] <描述>`）。
 4. 阶段性成果须产出可运行的 Demo 或可验证的输出样例。
 5. 遇到阻塞项（API 不可用、依赖冲突等），记录到 `docs/blockers.md` 并给出备选方案。
 
 ### 会话上下文复用规则
 
 - 每次新会话启动时，读取 `agents.md` 确定当前阶段。
 - 加载当前阶段 spec 文件，读取 checklist 恢复进度。
 - 上一会话遗留的未解决问题记录在 `docs/blockers.md`，须优先处理。
