 # Phase 3 — 前端小程序交互（P0）
 
 > **目标**: 完成微信小程序前端页面开发，实现文案录入、赛道选择、5 维评分展示、优化建议交互全流程。
 > **依赖**: Phase 2（AI 分析 API 端点可用）
 > **产出**: 可运行的小程序端代码 + 完整的用户交互流程
 
 ---
 
 ## 技术决策
 
 | 选项 | 选定 | 理由 |
 |------|------|------|
 | 框架 | **UniApp (Vue 3 + Vite)** | 一份代码多端编译，支持发布到微信小程序 |
 | UI 组件库 | **uView Plus 或 View Design** | 微信小程序友好，表单 / 评分组件丰富 |
 | 状态管理 | **Pinia** | Vue 3 生态标准 |
 | HTTP 请求 | **uni.request 封装** | UniApp 内置 |
 | 图表 | **ucharts** | 微信小程序兼容的图表库 |
 
 ---
 
 ## 任务清单

 - [ ] 使用 `npx degit dcloudio/uni-preset-vue#vite` 初始化 UniApp 项目

 - [ ] 配置 `pages.json`（注册所有页面路由）
 - [ ] 安装并配置 uView Plus 组件库

 - [ ] 配置全局样式变量（主题色、字体、间距体系）
 
 ### 3.2 首页 — 文案录入页面（`pages/index/index`）
 - [ ] 设计页面布局：上方输入区 + 赛道选择 + 底部提交按钮
 - [ ] 实现文本粘贴输入框（`<textarea>`，占位提示文案）
 - [ ] 实现实时字数统计（显示当前字数 / 5000）
 - [ ] 实现超 5000 字截断提示 + 分段建议 Toast
 - [ ] 实现赛道分类单选框（横向排列）：小红书种草、电商商品文案、本地文旅文案、短视频脚本
 - [ ] 实现图片上传按钮（配合 OCR，Phase 4 对接，本期占位）
 - [ ] 实现「AI一键分析」提交按钮（loading 状态 + 防重复点击）
 - [ ] 实现表单校验（空文本/超长/无赛道选择的错误提示）
 
 ### 3.3 分析结果页面（`pages/result/result`）
 - [ ] 设计结果页布局：顶部总体评分区域 + 5 维度 Tab 切换 + 优化建议区域
 - [ ] 实现总体评分环形进度条（S/A/B/C 对应不同颜色）
 - [ ] 实现 5 维度评分卡片：标题吸引力 / 情绪词与风格 / 卖点结构 / 人群匹配 / 综合流量分
 - [ ] 每个维度卡片展示：分数（数字） + 进度条 + 简要文字结论
 - [ ] 实现 Tab 切换（横向滑动或点击切换各维度详情）
 - [ ] 每个维度详情页展示：
   - 检测到的具体要素列表（如标题中的数字词、问句、利益词）
   - 缺失引流要素标注
   - 对应赛道标杆数据对比（如有）
 - [ ] 实现情绪词云展示（正向 / 共情 / 焦虑词高亮显示）
 
 ### 3.4 优化建议页面（`pages/result/result` 下部或独立子路由）
 - [ ] 实现「优化建议」卡片区域，分模块展示：

 ## 验收标准
 
 1. 首页文本输入框正常输入，实时字数统计准确
 2. 选择赛道后赛道标签高亮
 3. 未填写文本 / 未选赛道点击「AI一键分析」弹出友好提示不崩溃
 4. 提交后进入 loading 状态，按钮置灰不可重复点击
 5. 分析结果页展示 5 维度评分卡片（含分数+进度条+结论文字）
 6. Tab 切换各维度详情页正常展示检测要素列表
 7. 优化建议区域展示 3 套改写标题、段落调整、情绪词、卖点重构
 8. 点击优化建议的「复制」按钮成功复制到剪贴板
 9. 网络异常时展示错误页并提供「重试」按钮
 10. 超长文案（>5000 字）提示截断或分段建议
 11. 整体 UI 风格统一、无样式错乱、适配 iPhone/Android 不同屏幕
 12. 结果页返回首页数据不丢失（返回后重新获取）
